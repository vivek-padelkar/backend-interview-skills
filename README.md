# Backend Interview Skill

**Full Node.js backend engineer mock interview with real-time feedback.**

## Quick Start

### Invoke the skill:
```bash
/backend-interview
```

### Or use natural triggers:
- "start backend interview"
- "mock interview node"
- "practice MANG interview"
- "backend coding interview"

## What You Get

1. **Personalized Interview** — Select experience level, topic, difficulty, and company style
2. **5-8 Scenario Questions** — Dynamically generated based on your choices
3. **Immediate Feedback** — Grade + actionable feedback after each answer
4. **Comprehensive Report** — Score breakdown, strengths, gaps, and study plan
5. **Learning Resources** — YouTube, LinkedIn, docs, and must-read articles curated for your topic

## Experience Levels

- **Junior (0–2 years)** — Entry-level, learning core concepts
- **Mid (3–5 years)** — Comfortable with architecture, debugging
- **Senior (6–10 years)** — System design, trade-offs, mentoring
- **Staff (10+ years)** — Cross-team architecture, org-level impact

## Topics Covered

- Node.js Core (event loop, streams, buffers, workers, clustering)
- Express/Fastify (middleware, routing, error handling, performance)
- Databases (connection pooling, transactions, indexing, N+1 queries)
- System Design (scaling, rate limiting, caching, message queues)
- Microservices (service boundaries, communication, resilience)
- Security (JWT, OAuth, SQL injection, CORS, secrets management)
- Performance (profiling, memory leaks, load balancing, optimization)
- Testing (unit, integration, E2E, mocking, debugging tests)

## Company Interview Styles

- **Amazon** — STAR format, leadership principles, scale-focused
- **Google** — Algorithm depth, distributed systems, rigorous design
- **Meta** — Product at scale, real-time systems, pragmatic speed
- **Netflix** — Resilience, chaos engineering, streaming challenges
- **General MANG** — Balanced mix of all company styles

## Difficulty Levels

- **Easy** — Foundational concepts, single-service thinking
- **Medium** — Practical decisions, multi-service basics
- **Hard** — Deep trade-offs, production edge cases

## Tips for Best Results

1. **Think out loud** — Explain your reasoning; don't aim for perfection
2. **Mention trade-offs** — Real interviews reward nuance ("It depends on...")
3. **Ask clarifying questions** — Just like in real interviews
4. **Be specific** — Use tool names, code patterns, concrete examples
5. **Estimate first** — For system design, rough numbers matter more than perfect math

## Output

### After each question:
- Grade: ✅ Strong | 🟡 Partial | ❌ Needs Work
- Actionable feedback (2–3 lines)

### End-of-session report:
- Per-question scores and breakdown
- Strengths and confidence builders
- Gaps to close with priority ranking
- Top 3 study concepts
- Curated learning resources (YouTube, LinkedIn, docs, articles, GitHub repos)
